# 가챠 시뮬레이터

작업 중단. 유니티로 작업하기 적합하지 않다고 생각하여 리액트로 웹프로그래밍으로 넘어감.

## 신경 쓴 부분

1. Json을 이용한 저장 및 불러오기 구현
2. Json을 이용한 멀티랭귀지 구현
3. 자유로운 InputField 추가 및 데이터와의 바인딩
4. 싱글톤 대신 정적 변수와 함수를 활용하되 최소화하려고 함.

## 부족한 점

1. 데이터 바인딩이 깔끔하지 못하다.
2. InputField 업데이트 시 반영하는 부분을 구현하지 않았음.
3. 캔버스 내 UI 버튼이 많아서인지 onClick부분과 관련해 성능저하가 올 수 있다는 경고메시지가 뜸. 수정하다가 아예 클릭 안되는 상태로 방치함. 고칠 순 있는데 다른 플랫폼으로 갈아타는게 나을 것 같아 이대로 둠.(코드 상 문제는 아니고 유니티 컴포넌트 상 문제)

## 고민한 부분

1. 데이터베이스랑 연동할 때 InputField의 값을 어떻게 업데이트 하는 것이 최적화될 지 고민하다가 구현하지 못했다. 이벤트는 활용할 수 있지만 매우 잦은 업데이트가 최적화에 문제를 주지 않으려면 단계를 최소화해야할 것 같다.
2. 아무래도 이러한 복잡한 UI는 체계적으로 구성해야할 것 같다. 이를 돕는 UI Toolkit이 있는데 아직 기능이 많이 부족한 것 같다. 윈폼에 DatagridView와 같은 기능이 없어서 직접 구현해야할 듯.
